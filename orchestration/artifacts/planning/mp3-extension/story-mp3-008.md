# Story mp3-008: Waveform Display

## Summary
Build the `WaveformDisplay` canvas component that visualizes audio waveform peaks for MP3 tracks, with a real-time playhead and click-to-seek interaction.

## Background
Waveform visualization is a core DJ feature that shows the track structure at a glance. The waveform is generated by downsampling the decoded `AudioBuffer` into ~800 amplitude peak values. The component renders these as vertical bars on an HTML5 canvas, with a playhead that moves in real-time. Users can click on the waveform to seek to any position.

## Technical Notes
- `extractWaveformPeaks`: divide `buffer.getChannelData(0)` into N buckets, take max absolute value per bucket. Use `Float32Array` view directly -- no copies needed. Default 800 points.
- Canvas rendering: use `requestAnimationFrame` only for the playhead animation. The static waveform bars can be drawn once to an off-screen buffer and then composited on each frame with the playhead on top.
- Alternatively, redraw the entire canvas each frame (simple, and fast enough for ~800 bars at 60fps).
- Use `canvas.getContext('2d')` -- no WebGL needed.
- The `onSeek` callback should call `playerRegistry.get(deckId)?.seekTo(time)`.
- Ghost playhead uses `onMouseMove` to track cursor X position.

### Edge cases
- Very short track (< 5 seconds): fewer bars than canvas width. Each bar may be wider. Handle by adjusting bar width.
- Very long track (60+ minutes): ~800 peaks still works, each peak covers more audio. No performance issue.
- Canvas resize (window resize): redraw waveform. Use `ResizeObserver` or redraw on width change.
- Track with silence (all zeros): bars at 0 height -- acceptable, shows "flat line"
- `waveformData` is null while `isAnalyzing` is false and no track loaded: show empty state
